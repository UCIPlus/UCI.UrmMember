# 产品上下文

## 产品定位

UCI Member 是优迪邦智能洗车平台的用户端应用，为消费者提供便捷的智能洗车服务体验。

## 解决的问题

### 用户痛点

1. **洗车不便**：传统洗车需要排队等待，耗时费力
2. **价格不透明**：传统洗车价格波动大，缺乏标准
3. **券证管理困难**：纸质优惠券容易丢失，使用不便
4. **充值不灵活**：传统充值卡只能在特定门店使用

### 技术债务（原系统）

1. **性能问题**：PHP View 渲染方式响应慢，用户体验差
2. **维护困难**：前后端耦合严重，修改困难
3. **扩展受限**：难以适配多端（小程序、H5）
4. **开发效率低**：每次修改需要部署整个后端

## 核心功能

### 1. 个人中心

**目标**：用户信息和资产概览的统一入口

**功能点**：

- 用户信息展示（昵称、头像、会员状态）
- 资产统计（自助洗车券、自动洗车券、核销券、账户余额）
- 快捷入口（组合卡购买、各类券管理）
- 功能菜单（消费记录、推荐返佣、站点查询等）

**用户价值**：一目了然地查看自己的资产和快速访问各项功能

### 2. 优惠券管理

#### 2.1 自助洗车券

**目标**：管理用于自助洗车设备的优惠券

**功能点**：

- 券列表展示（数量、有效期、状态）
- 券详情查看
- 使用记录追踪

**用户价值**：随时查看和管理自助洗车券，避免过期浪费

#### 2.2 自动洗车券

**目标**：管理用于自动洗车设备的优惠券

**功能点**：

- 券列表展示（数量、有效期、状态）
- 券详情查看
- 使用记录追踪

**用户价值**：掌握自动洗车券的使用情况

#### 2.3 核销券

**目标**：管理可在指定门店核销的优惠券

**功能点**：

- 券列表展示
- 核销规则查看
- 核销记录追踪
- 扫码核销

**用户价值**：便捷使用和管理核销券，支持多种商品兑换

### 3. 组合购券充值

**目标**：提供灵活的充值和购券方案

**功能点**：

- 套餐列表展示（不同面值、优惠力度）
- 套餐详情（包含券种类、数量、有效期）
- 购买流程（选择套餐 → 支付 → 发券）
- 订单管理（查看购买历史）
- 支付方式（微信支付、支付宝支付、余额支付）

**用户价值**：

- 批量购券更优惠
- 灵活组合满足不同需求
- 充值即送券，性价比高

### 4. 消费记录

**目标**：透明化消费历史，便于用户追溯

**功能点**：

- 自助洗车记录（时间、地点、设备、消费金额）
- 自动洗车记录（时间、地点、设备、服务类型）
- 记录详情（设备信息、服务时长、费用明细）
- 按时间筛选

**用户价值**：

- 清楚了解每一笔消费
- 方便对账和问题追溯
- 形成消费习惯数据

### 5. 推荐返佣

**目标**：激励用户推荐新用户，实现裂变增长

**功能点**：

- 推荐链接/二维码生成
- 推荐记录（推荐人数、返佣金额）
- 佣金提现（申请提现、提现记录、到账状态）
- 冻结佣金管理（待结算佣金）

**用户价值**：

- 推荐朋友获得现金返佣
- 佣金可提现或用于消费
- 形成社交传播

### 6. 设备扫码

**目标**：快速启动洗车服务

**功能点**：

- 扫描设备二维码
- 设备信息展示
- 选择使用券或付费
- 启动服务

**用户价值**：

- 扫码即洗，无需等待
- 自主选择支付方式
- 全程自动化

### 7. 站点查询

**目标**：帮助用户找到附近的洗车点

**功能点**：

- 地图展示所有站点
- 站点详情（地址、设备数量、营业时间）
- 导航功能
- 设备状态实时显示

**用户价值**：

- 快速找到最近的洗车点
- 了解设备可用情况
- 规划洗车路线

### 8. 营销活动

**目标**：通过活动提升用户活跃度和消费频次

**功能点**：

- 签到领券
- 砍价活动
- 秒杀活动
- 分享领券
- 消费满赠

**用户价值**：

- 获得更多优惠
- 趣味互动体验
- 额外福利

## 用户旅程

### 新用户首次使用

1. 扫描洗车设备二维码 → 引导关注公众号/打开小程序
2. 授权登录 → 自动注册
3. 查看首次优惠 → 参与新人活动领取优惠券
4. 选择付费或使用券 → 启动洗车
5. 洗车完成 → 收到提示，引导分享或购买套餐

### 老用户日常使用

1. 打开小程序 → 查看个人中心
2. 扫码启动洗车 → 选择券或付费
3. 查看消费记录 → 了解消费情况
4. 券不足时购买组合卡 → 完成充值
5. 推荐朋友获取返佣 → 提现

## 产品原则

### 1. 简单易用

- 核心流程不超过 3 步
- 关键信息一目了然
- 减少用户思考和操作

### 2. 快速响应

- 页面加载时间 < 1 秒
- 操作反馈即时
- 优化网络请求

### 3. 透明可信

- 所有消费记录可追溯
- 券状态实时同步
- 退款规则明确

### 4. 激励分享

- 推荐有奖
- 分享领券
- 社交传播

## 与原系统的对比

| 维度 | 原系统（PHP View） | 新系统（uni-app） |
|------|-------------------|-------------------|
| **架构** | 前后端耦合 | 前后端分离 |
| **响应速度** | 较慢（服务器渲染） | 快速（客户端渲染） |
| **多端支持** | 仅 H5 | 小程序 + H5 |
| **开发效率** | 低（需要部署后端） | 高（前端独立开发） |
| **维护性** | 差（耦合严重） | 好（模块化清晰） |
| **用户体验** | 一般 | 优秀（交互流畅） |
